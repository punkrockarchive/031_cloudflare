
Cloudflare Pages 表示確認と次工程可否について
============================================================

結論：
「“このページが見つかりません” が表示されない」＝正常の可能性が高い。
状況次第では解消不要。

次工程（/api/line/webhook 実装）で重要なのは：

・https://sawara-dpf.pages.dev/ が外部から到達できること
・https://sawara-dpf.pages.dev/api/line/webhook が POST を受けて 200 を返せること

トップページ表示そのものは本質ではない。


============================================================
現在の表示状況別の判断
============================================================

A. HelloWorld が表示されている
→ 問題なし（成功）。次工程へ進んでOK。

B. 何も表示されない／真っ白
→ 次工程前に一度確認推奨。
   ただし API が動けば Webhook は作れる可能性あり。

C. Cloudflare エラー画面（5xx / Access denied / Redirect loop 等）
→ 先に解消推奨。Webhookも失敗しやすい。


============================================================
次工程に進めるかの最短チェック
============================================================

1) トップ（静的確認）
https://sawara-dpf.pages.dev/

→ HelloWorld など何か表示されればOK

2) Webhookパス（到達性確認）
https://sawara-dpf.pages.dev/api/line/webhook

→ 現時点で 404 でもOK（未実装のため）
→ 重要なのはドメインに到達できること


============================================================
Direct Upload で 404 になる典型原因
============================================================

・index.html がルート直下に無い
  例：/myfolder/index.html になっている

→ Pages は基本的に /index.html を探す


確認ポイント：

・Deployments タブで最新デプロイが Success か
・アップロードZIPの直下に index.html があるか


============================================================
まとめ
============================================================

・「ページが見つかりません」が表示されないこと自体は問題ではない
・HelloWorld が表示されていれば成功
・次工程に必要なのは Webhook URL が 200 を返せる状態
・Pages Functions で /api/line/webhook を実装し、curl で 200 を確認すればOK
